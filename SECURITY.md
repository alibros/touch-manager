# Security policy

Touch Manager controls a hardware-writing process, so validation bypasses, command
injection, unsafe target selection, catalog-trust failures, and destructive device
behavior are security issues.

Please do not disclose a suspected vulnerability in a public issue. Use the repository's
private GitHub Security Advisory form after it is enabled. Include the affected version,
reproduction steps, impact, and whether a physical device was involved.

The project is currently pre-release. Security fixes target the latest source version;
there is not yet a separately maintained stable branch.
